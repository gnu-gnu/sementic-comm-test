# 용어 및 개념 정리

이 시뮬레이션에서 사용되는 학술 용어들의 정의와, 수치가 높거나 낮을 때의 의미를 정리한다.

---

## 채널 관련

### SNR (Signal-to-Noise Ratio, 신호대잡음비)

신호 전력과 잡음 전력의 비율. 단위는 dB.

```
SNR(dB) = 10 × log₁₀(신호 전력 / 잡음 전력)
```

| 값 | 의미 |
|----|------|
| 20dB | 신호가 잡음의 100배 — 깨끗한 채널 |
| 10dB | 신호가 잡음의 10배 — 보통 |
| 0dB | 신호와 잡음이 같음 — 열악 |
| -10dB | 잡음이 신호의 10배 — 극한 환경 |

**높을수록 좋다.** 실제 환경: 5G 실내(20~30dB), 셀 경계(0~5dB), 재난 환경(마이너스 가능).

### AWGN (Additive White Gaussian Noise)

채널에 추가되는 백색 가우시안 잡음. 가장 기본적인 채널 모델.
"Additive"는 원래 신호에 잡음이 더해진다는 의미이고, "White"는 모든 주파수 대역에 균등하게 분포한다는 의미이며, "Gaussian"은 잡음의 진폭이 정규분포를 따른다는 의미.

### BER (Bit Error Rate, 비트 오류율)

전송된 비트 중 오류가 발생한 비트의 비율.

```
BER = 오류 비트 수 / 전체 전송 비트 수
```

| 값 | 의미 |
|----|------|
| 0 | 에러 없음 — 완벽한 전송 |
| 1e-6 | 백만 비트 중 1개 오류 — 매우 양호 |
| 1e-3 | 천 비트 중 1개 오류 — JPEG 디코딩 실패 가능 |
| 1e-1 | 10비트 중 1개 오류 — 데이터 완전 손상 |
| 0.5 | 랜덤 추측과 동일 — 정보 전달 불가 |

**낮을수록 좋다.** JPEG 같은 압축 포맷은 BER이 1e-4 이상이면 디코딩이 실패할 수 있다.

### Rayleigh Fading

다중경로 전파 환경에서 발생하는 페이딩 모델. 직접 경로(Line of Sight)가 없는 환경에서 신호 세기가 시간에 따라 크게 변동하는 현상. 이 시뮬레이션에서는 사용하지 않았으며, AWGN만 적용했다.

## 변조 관련

### Modulation (변조)

디지털 비트를 아날로그 신호(심볼)로 변환하는 과정.

| 방식 | bits/symbol | 특성 |
|------|-------------|------|
| BPSK (Binary Phase Shift Keying) | 1 | 가장 강건, 가장 느림 |
| QPSK (Quadrature PSK) | 2 | BPSK 대비 2배 빠름, 비슷한 에러율 |
| 16-QAM (Quadrature Amplitude Modulation) | 4 | 4배 빠르지만 에러에 취약 |

bits/symbol이 **높을수록 전송 속도가 빠르지만, 동일 SNR에서 에러율이 높아진다.**

### Spectral Efficiency (주파수 효율)

단위 대역폭(Hz)당 전송 가능한 비트 수. 단위는 bps/Hz.

```
Spectral Efficiency = bits/symbol × code rate
```

| 구성 | Spectral Efficiency |
|------|-------------------|
| BPSK + Hamming(7,4) | 1 × 4/7 = 0.57 bps/Hz |
| QPSK + Hamming(7,4) | 2 × 4/7 = 1.14 bps/Hz |
| 16-QAM + Hamming(7,4) | 4 × 4/7 = 2.29 bps/Hz |

**높을수록 같은 대역폭으로 더 많은 데이터를 전송할 수 있다.** 하지만 높은 spectral efficiency를 안정적으로 사용하려면 높은 SNR이 필요하다.

### Adaptive Modulation and Coding (AMC)

채널 상태에 따라 변조 방식과 부호화율을 동적으로 변경하는 기술. 채널이 좋으면 16-QAM으로 빠르게, 채널이 나쁘면 BPSK로 안전하게 전송한다. LTE/5G에서 핵심적으로 사용된다.

## 채널 부호화 관련

### Channel Coding (채널 부호화)

전송 전 데이터에 중복 비트(redundancy)를 추가하여 수신 측에서 에러를 검출/정정할 수 있게 하는 기술.

### Hamming(7,4)

이 시뮬레이션에서 사용한 채널 부호화 방식.
- 4비트 데이터 → 7비트 부호어 (3비트 패리티 추가)
- **Code rate**: 4/7 ≈ 0.571 (데이터 57.1%, 보호 비트 42.9%)
- **1비트 에러 정정 가능**, 2비트 이상은 정정 불가
- 간단하지만 실전에서는 더 강력한 코드(LDPC, Turbo, Polar)를 사용

### Code Rate (부호화율)

원본 데이터 비트 수 대비 부호화된 전체 비트 수의 비율.

```
Code rate = 원본 비트 / 부호화 비트
```

| 값 | 의미 |
|----|------|
| 1.0 | 부호화 없음 — 보호 없음 |
| 4/7 ≈ 0.57 | Hamming(7,4) — 중간 보호 |
| 1/2 = 0.50 | 데이터 절반이 보호 비트 — 강한 보호 |
| 1/3 ≈ 0.33 | 매우 강한 보호, 전송 효율 낮음 |

**높을수록 전송 효율이 좋지만 에러 정정 능력이 약하다.**

## 소스 부호화 관련

### Source Coding (소스 부호화)

원본 데이터의 중복을 제거하여 압축하는 과정. 이 시뮬레이션에서는 JPEG을 사용.

### JPEG

DCT(이산 코사인 변환) 기반 이미지 손실 압축. 허프만 부호화로 비트스트림을 생성한다.
비트스트림 구조에 의존하기 때문에, **한 비트의 오류가 이후 전체 디코딩을 연쇄적으로 실패시킬 수 있다** — 이것이 cliff effect의 직접적 원인이다.

### Compression Ratio (압축률)

원본 대비 압축된 데이터의 크기 비율.

```
압축률 = 원본 크기 / 압축 크기
```

이 시뮬레이션에서의 압축률:

| 방식 | 크기 | 압축률 |
|------|------|--------|
| Raw (비압축) | 28.3 MB | 1x |
| JPEG (q=85) | 1.1 MB | 25x |
| JSCC (16ch bottleneck) | 630 KB | 46x |
| 캡션 (BLIP) | 99 B | 299,869x |

## 품질 평가 지표

### PSNR (Peak Signal-to-Noise Ratio, 최대 신호대잡음비)

원본과 복원 이미지의 픽셀 단위 차이를 측정. 단위는 dB.

```
PSNR = 10 × log₁₀(MAX² / MSE)
```

| 값 | 의미 |
|----|------|
| ∞ | 원본과 동일 |
| 40dB 이상 | 육안으로 구분 불가 |
| 30~40dB | 미세한 차이 |
| 20~30dB | 열화 보이지만 내용 파악 가능 |
| 20dB 이하 | 심하게 깨짐 |

**높을수록 좋다.** 다만 픽셀 단위 비교라서 사람의 시각 인지와 다를 수 있다.

### SSIM (Structural Similarity Index)

사람의 시각 인지에 맞춰 밝기, 대비, 구조 패턴 세 가지를 종합 비교하는 지표.

| 값 | 의미 |
|----|------|
| 1.0 | 완벽히 동일 |
| 0.9 이상 | 거의 구분 불가 |
| 0.7~0.9 | 구조는 보이지만 열화 있음 |
| 0.5 이하 | 심하게 왜곡 |

**높을수록 좋다.** PSNR보다 사람이 느끼는 품질과 더 가깝다.

### MSE (Mean Squared Error)

원본과 복원 이미지 간 픽셀 값 차이의 평균 제곱. PSNR 계산의 기반이 된다.

**낮을수록 좋다.**

## 시멘틱 통신 관련

### JSCC (Joint Source-Channel Coding)

소스 부호화(압축)와 채널 부호화(에러 보호)를 하나의 DNN으로 **통합 학습**하는 방식. 전통적인 분리 설계(Shannon의 분리 정리)와 대비되는 접근.

장점:
- 채널 상태에 맞는 인코딩을 자동으로 학습 (Adaptive)
- Cliff effect 없이 graceful degradation
- 변조 방식 선택이 필요 없음

한계:
- 모델 용량에 의해 최대 품질이 제한됨
- 학습 데이터에 의존 (과적합 가능)
- 인코딩/디코딩에 GPU 연산 필요

### Cliff Effect (절벽 효과)

전통적 통신에서 채널 품질이 일정 임계점 이하로 떨어지면 품질이 갑자기 0으로 붕괴하는 현상.
소스 부호화와 채널 부호화가 분리 설계되어 있기 때문에, 채널 부호화의 에러 정정 한계를 넘으면 소스 디코딩이 완전히 실패한다.

### Graceful Degradation (점진적 열화)

JSCC에서 채널 품질이 나빠져도 품질이 점진적으로 떨어지는 현상.
DNN이 연속 신호 공간에서 인코딩하므로, 노이즈가 커져도 "일부만 열화"될 뿐 전체가 붕괴하지 않는다.

### Power Normalization (전력 정규화)

전송 신호의 평균 전력을 1로 정규화하는 과정. JSCC에서 인코더 출력에 적용한다.
이를 하지 않으면 모델이 신호 크기를 키워서 노이즈를 상대적으로 줄이는 "치팅"을 할 수 있기 때문에, 공정한 비교를 위해 필수적이다.

### Semantic Fidelity (의미적 충실도)

전송된 메시지의 의미가 원본의 의미와 얼마나 일치하는지를 나타내는 개념.
전통적 통신의 bit-level fidelity와 대비되며, 아직 표준화된 정량 지표가 없는 연구 과제이다.

### Hallucination (환각)

AI 모델이 입력 데이터에 없는 정보를 생성하는 현상. 이 시뮬레이션에서 BLIP 캡셔닝 모델이 이미지에 없는 주소와 날짜를 생성한 것이 대표적인 예시다. 시멘틱 통신에서는 의미의 정확성이 곧 통신의 정확성이므로, hallucination은 통신 에러와 동등한 문제가 된다.

## Shannon의 정리와 시멘틱 통신의 관계

### Shannon의 분리 정리 (Separation Theorem)

소스 부호화와 채널 부호화를 **분리하여 각각 최적화해도**, 블록 길이가 무한대일 때 **전체적으로 최적**이라는 정리. 현대 통신 시스템(LTE, 5G)은 이 원리에 기반한다.

그러나 현실에서는:
- 블록 길이가 유한하다
- 지연 제약이 있다
- 채널 상태가 시변한다

이런 제약 하에서 **분리 설계가 최적이 아닐 수 있으며**, JSCC와 같은 통합 설계가 더 나은 성능을 보일 수 있다. 이것이 시멘틱 통신 연구의 이론적 근거 중 하나이다.
